Learning JavaScript

HTML, CSS, and JavaScript

HTML is like the script of a play. It decides which characters appear on stage and what props are present.

CSS is like the stage design and costumes. It gives the styling of the scene depending on what the designer wants to present.

JavaScript is like the actors. They move around, deliver lines, and sometimes engage with the audience.

Without JavaScript, your website might look pretty, but it won’t do anything.

Example

Button in HTML:

<button id="doorbell">Ring Doorbell</button>

CSS to style it:

#doorbell {
  background-color: skyblue;
  padding: 10px;
  font-size: 16px;
}

JavaScript to make it do something:

document.getElementById("doorbell").addEventListener("click", () => {
  alert("Ding dong! Someone’s at the door");
});

When you click the button, JavaScript makes the doorbell ring!

Control Flow and Loops

Control Flow is the order in which things happen.

Here’s an example of a loop. Imagine making cocktails for guests at a party:

Start: Put ice in the cocktail shaker.

Loop: For each person

  • Add the required alcohol in the shaker.
  • Add the mixer.
  • Pour it into the glass.

End: Once you’ve gone through all the guests, you stop.

That’s exactly how a loop works: repeating steps until you’re done.

JavaScript version

let guests = ["Matthew", "Dawn", "Gary"];

for (let guest of guests) {
  console.log("Make cocktail for " + guest);
}

Output

Make cocktail for Matthew
Make cocktail for Dawn
Make cocktail for Gary

What is the DOM?

The DOM (Document Object Model) is like a living map of your webpage. Every heading, image, and button is a “branch” on the tree called a node.

With JavaScript, navigate the tree, grab a node, and change it. For example (the document is the top of the page’s node tree under the “window”): 

document.querySelector("h1").textContent = "Hello, World!";

That one line of code changes the text of the <h1> heading.

Arrays vs. Objects

Arrays are like numbered shopping lists.

let shoppingList = ["milk", "bread", "apples"];
console.log(shoppingList[0]); // milk

You get items based on their number starting from 0.

Objects are like labeled boxes.

let dog = { name: "Frank", breed: "Jack Russell", age: 2 };
console.log(dog.name); // Frank

You don’t care about the order — you grab things by their label (name, breed, age).

Functions

A function is like a recipe. Instead of writing out the whole recipe every time you want to make a cocktail, you keep the instructions in one place and reuse them. 

function makeCocktail(name) {
  console.log("Here’s a cocktail for " + name);
}

makeCocktail("Matthew"); // Here’s a cocktail for Matthew
makeCocktail("Dawn");    // Here’s a cocktail for Dawn

Why are functions helpful? Because they save time, keep your code tidy, and let you reuse logic over and over again.
